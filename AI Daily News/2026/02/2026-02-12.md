#### OpenAI, Skills API + Shell Tool + Server-side Compaction 발표 — MCP 대안 에이전트 스킬 표준
[DEV] [PM] | **소스**: [OpenAI upgrades Responses API with Skills and Shell](https://venturebeat.com/orchestration/openai-upgrades-its-responses-api-to-support-agent-skills-and-a-complete) | **관련 제품**: [[openai/openai|OpenAI]]
**태그**: `#agent-skills` `#agent-runtime` `#protocols`

OpenAI가 2월 11일 Responses API에 Skills, Hosted Shell, Server-side Compaction 기능을 추가했다. Skills는 SKILL.md 매니페스트를 기반으로 파일 번들(인스트럭션+스크립트+에셋)을 재사용 가능한 모듈로 패키징하는 시스템이다. Shell Tool은 Debian 12 기반 컨테이너(Python 3.11, Node.js 22, Java 17, Go 1.23 등)에서 에이전트가 코드를 생성·실행하고 네트워크 접근까지 가능하다. Server-side Compaction은 긴 대화의 컨텍스트를 자동 압축하여 장기 실행 에이전트의 안정성을 높인다. 이는 Anthropic의 Claude Code/MCP 생태계와 직접 경쟁하는 에이전트 인프라 전략으로, 에이전트 스킬 설계 패턴에 중요한 참조점이 된다.

**KonaChain 임팩트:**
- 🏗️ **아키텍처**: 우리 MCP 기반 tool 연결과 직접 경쟁하는 패턴. Skills는 파일 번들 방식으로 MCP의 JSON-RPC 서버 방식과 근본적으로 다름. 양쪽 호환 어댑터 필요성 검토 대상.
- 📊 **경쟁**: Salesforce, SAP 등 경쟁사가 OpenAI Skills를 채택하면 MCP-only 전략이 리스크가 됨. 듀얼 지원 필요성 증가.
- ⚠️ **리스크**: Shell Tool의 서버 실행 모델은 우리 on-premise ERP 연동과 보안 모델 충돌 가능.

#### Z.ai GLM-5: 744B 파라미터 오픈소스 모델 출시 — 에이전트 백엔드 비용 최적화 선택지 확대
[DEV] [LEAD] | **소스**: [z.ai's open source GLM-5 achieves record low hallucination rate](https://venturebeat.com/technology/z-ais-open-source-glm-5-achieves-record-low-hallucination-rate-and-leverages) | **관련 제품**: —
**태그**: `#open-source` `#agent-runtime`

Zhipu AI(Z.ai)가 2월 11일 GLM-5를 출시했다. 744B 파라미터(MoE, 40B active) 규모로 Huawei 칩에서 훈련되었으며 MIT 라이선스로 공개되었다. AA-Omniscience Index에서 -1점(전작 대비 35점 향상)을 달성하여 OpenAI, Google, Anthropic을 포함한 전체 AI 업계 1위 지식 신뢰도를 기록했다. BrowseComp 오픈소스 1위(75.9), Humanity's Last Exam에서 Claude Opus 4.5 능가(50.4점). DeepSeek Sparse Attention을 통합하여 배포 비용을 절감하면서 200K 토큰 컨텍스트 윈도우와 131K 토큰 출력을 지원한다. 가격은 입력 $0.80-1.00/M, 출력 $2.56-3.20/M으로 경쟁력이 높다. OpenRouter와 HuggingFace에서 즉시 이용 가능하며, Vercel AI Gateway에도 동시 등록되었다.

**KonaChain 임팩트:**
- 💰 **비용**: Claude/OpenAI 대비 60-70% 비용 절감 가능성. 200K 컨텍스트 윈도우는 ERP 데이터 분석 에이전트에 충분한 규모. 한국 중소기업 대상 가격 경쟁력 확보에 활용 가능.
- 🏗️ **아키텍처**: Deep Agent의 LLM 다중 백엔드 전략에서 비용 최적화 시나리오의 유력 후보.
- ⚠️ **리스크**: Huawei 칩 훈련 → 미중 기술 규제 리스크. 한국 공공기관 납품 시 제약 가능.
