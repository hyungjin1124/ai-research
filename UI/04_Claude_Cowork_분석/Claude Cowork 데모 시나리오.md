    - (2) 은행대사(매칭/미매칭/중복)
        
    - (3) 결산 체크리스트 업데이트

- 에이전트가 다음을 자동 생성:

    - “중복 지급 의심 1건” 자동 하이라이트
        
    - “미식별 입금 1건” → 확인 필요(거래처/근거) 티켓화 제안
        
    - “차지백/수수료/이자” → 계정과목 자동 추천
        

#### Step 4. 조정분개(JE) 초안 생성 + 승인 요청(통제)

- 에이전트가 `proposed_journal_entries.xlsx` 생성:

    - 미식별 입금: **가수금** 처리 후보 + 확인 후 정리 안내
        
    - 차지백: **매출차감/기타비용** 후보
        
    - 수수료/이자: **은행수수료/이자수익** 자동 분개안
        
- 사용자에게 “전표 반영(생성/내보내기)” 같은 중요 액션은 **승인**을 요청하도록 연출

# 데모 시나리오 - 항공권 정보 조사

#### 목적
 * Claude in Chrome 커넥터를 통해 웹 사이트에 접속하여 폼을 입력한 후 정보를 조사하여 최종 결과물로 정리

### Prompt
#### Step 1: 항공권 가격 비교 분석 리포트 작성 요청
Chrome을 사용해서 스카이스캐너 웹사이트에 접속하고, 5월 첫째 주와 둘째 주 사이에 도쿄 여행을 3박 4일로 가는 경우 어떤 날짜에 어떤 항공권이 저렴한지 비교 분석하여 리포트를 작성해줘.

1. 항공권 예약 페이지로 이동
2. 검색 조건 입력:
    - 여정 유형: 왕복
    - 출발지: 서울/인천 (ICN)
    - 도착지: 도쿄/나리타 (NRT)
    - 탑승객: 성인 1명
    - 좌석 등급: 일반석
3. 예약 진행하지 말고 결과물만 작성해줘. 항공편 선택이랑 결제는 내가 직접 할게.

#### Step 2: 날짜 변경
6월 첫째 주와 둘째 주 사이 날짜로 변경하여 조사해주세요.
